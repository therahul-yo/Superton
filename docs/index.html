§ 01 / installation

Three doors into the palace.

Pick the one that fits your machine. None require a cloud account. None phone home.

door 01 · recommended

via uv

Single command. Always uses an isolated environment.

$uv tool install 'git+https://github.com/therahul-yo/Superton.git'copy
$superton initcopy
Requires uv · install uv
door 02 · hackers

from source

Clone, edit, contribute. Great if you want to tune the Modelfile.

$git clone https://github.com/therahul-yo/Superton.gitcopy
$cd Superton && uv run superton initcopy
Python 3.11+ · Apache-2.0
door 03 · prerequisite

the model runtime

SuperTon answers locally via Ollama. init can install this for you, or do it yourself.

$brew install ollamacopy
$curl -fsSL https://ollama.com/install.sh | shcopy
macOS · Linux · Windows (WSL)

The palace remembers in your voice — not a frontier model's.

§ 04 / design tenets

Three laws of the palace.

Opinionated trade-offs we made early and won't apologize for.

I.
Verbatim beats summarized

A summary loses the half-remembered phrase you'll actually search for in six months. Drawers store the original text. Storage is cheap; recall is everything.

II.
Refuse over confabulate

When the palace is silent, Miniton says so — and suggests the closest files. No hallucinated dates, names, or numbers. Trust over confidence.

III.
Local until proven otherwise

No API keys. No telemetry. No SaaS dependency. The palace lives on your laptop; the model lives in your Ollama. If you go offline, nothing breaks.
